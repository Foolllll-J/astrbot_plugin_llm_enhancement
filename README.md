## 📖 简介

一款为 [AstrBot](https://astrbot.app) 设计的 LLM 增强插件。通过集成智能唤醒、消息合并、多媒体解析、QQ 群成员信息注入与黑名单管理，并提供扩展的群管理 LLM 工具、跨会话定向发消息能力，以及可配置的管理员校验/危险工具开关/二次确认机制，提升 Bot 在群聊与私聊中的参与感、逻辑连贯性与安全性，让你的 Bot 不再只是被动响应，而是能够“听懂”话题、“看懂”视频、“感知”群成员并执行受控操作。

---

## ✨ 功能

本插件通过以下核心模块，将您的 Bot 升级为具备“直觉”与“多维感知”的智能实体：

* **🧠 智能唤醒系统**：支持相关性、活跃期、特定提及、专业答疑等多种触发模式，内置沉默与冷却机制。
* **📦 消息合并增强**：智能聚合短时间内多条消息，支持跨用户上下文合并、撤回实时监控、请求前快速拦截与兜底校验，以及合并数量上限控制。
* **🎬 媒体文件感知**：支持视频抽帧/ASR、JSON 卡片解析、文件引用感知与文本摘录注入、合并转发内容解析。支持聊天记录视频抽帧上限控制与占位符回退。内置视频总结缓存机制，避免重复解析。
* **🛠️ 增强管理工具**：集成通讯录查询、群成员查询、历史消息检索、群聊相关读写操作、跨会话消息发送、黑名单管理等 LLM 函数工具，形成“查询 -> 判断 -> 执行”的完整闭环，并支持管理员校验、危险工具开关与二次确认。
* **🛡️ 黑名单管理器**：支持拉黑/解除拉黑、分页查询与状态查询；支持三档黑名单拦截等级（仅 LLM / 指令+LLM / 所有消息）。

## 🚀 安装

1. **环境准备**：
   * 系统需安装 FFmpeg 并将其加入环境变量（用于视频/GIF 解析）。
   * 在插件目录执行 `pip install -r requirements.txt` 安装依赖。
2. **插件部署**：
   * 将 `astrbot_plugin_llm_enhancement` 文件夹放入 AstrBot 的 `plugins` 目录。
3. **重启 AstrBot**。

## 💡 使用

本插件旨在提供**全自动化**、**无感**的增强体验，以下是各模块详细的功能细节与配置参考：

### 1️⃣ 🧠 智能唤醒：从“被动回复”到“主动参与”

Bot 不再死板地等待艾特，而是根据语境决定是否介入：

* **唤醒判定**：
  * **前置唤醒判定**：可单独配置 Provider 和提示词，仅对普通 `@` / 唤醒前缀请求生效，用于模拟真人沉默场景；支持占位符 `{time}`、`{user_id}`、`{user_name}`、`{group_id}`、`{history}`、`{message}`、`{persona}`、`{random}`。
  * **活跃在线期**：Bot 回复后会进入短暂的“兴奋期”，此时用户的后续追问无需艾特即可直接响应。
  * **同用户延长**：可配置唤醒延长仅对“上次触发请求的用户”生效，避免窗口期被其他人消息打断。
  * **模型判定延长**：可选指定一个 Provider 对“是否继续响应”做 T/F 判定；支持单独配置唤醒延长判定提示词（占位符同上），判定不可用时自动回退本地相似度。
* **语义与提及触发**：
  * **相关性唤醒**：自动分析当前消息与上下文的关联度。当聊到与上文相关的话题时，它可以主动跟进。并且会对超短消息、问句等场景做降噪；还可按当前消息长度动态选择历史条数。
  * **专业答疑**：识别群内专业提问或求助信号，精准提供帮助。答疑判定为“提问门槛 + 语义评分”两段式，减少非问句误唤醒。
  * **无聊唤醒**：识别“冷场/无聊/求聊天”语境并触发唤醒；采用前置门槛过滤长叙述，降低误触发。
  * **正则提及唤醒**：支持 regex 规则。支持 `re:pattern` 与 `/pattern/flags`（flags 支持 `i,m,s,x,a,u,L`）；仅当规则既不以 `re:` 开头、也不符合 `/pattern/flags` 形式时，才按普通文本包含匹配。示例：`re:^(?=.*群主)(?=.*是)(?=.*笨蛋).*$`、`/^(?=.*群主)(?=.*是)(?=.*笨蛋).*$/i`。
* **约束与礼节**：
  * **唤醒屏蔽规则**：同样支持普通文本包含、`re:pattern` 与 `/pattern/flags`。命中后将阻止本次唤醒（bot 管理员不受影响）。
  * **沉默机制**：当用户表示厌烦或进行辱骂时，Bot 会根据配置的阈值自动“闭嘴”一段时间。
  * **空 @ 回复**：当用户空艾特时，插件会自动注入“某某@了你”提示，并补充该消息前最近 5 条群聊普通消息。
* **兜底机制**：
  * **主动插嘴**：在以上机制均未触发时，仍有极小概率主动参与聊天，模拟真人的随机性。

### 2️⃣ 📦 消息合并：聚合碎片输入

通过两种合并模式，解决碎片化输入导致的语义割裂与回复跳步问题：

* **模式选择**：
  * **动态软重算**：先发起请求；若响应返回前收到新消息，会放弃上一轮结果，并在下一轮重算上下文。
  * **硬等待合并**：在短窗口内先收集后续消息，再统一发起请求，有助于节省 token，但会导致平均响应时间变长。
* **窗口与边界控制**：
  * **统一规则**：两种模式都支持配置多人消息聚合、数量上限与后续消息唤醒要求，避免窗口期无限收集。
  * **撤回安全**：合并阶段支持实时撤回监听；命中后会立即剔除对应消息，必要时终止本次请求。
  * **最终校验**：请求与回复前会再次校验消息可用性；不可获取或已撤回消息会被过滤，全部失效时取消请求。

### 3️⃣ 🎬 理解增强：打破视觉次元壁

让 Bot 能看到听到富媒体消息的具体内容：

* **媒体文件感知**：
  * **私聊媒体唤醒**：私聊下直接发送视频/文件/转发/JSON 组件也可触发请求，无需额外文本。
  * **多维视频解析**：自动对视频进行抽帧与 ASR 语音转录，让 Bot 既能“看”又能“听”。
  * **高效抽帧**：采用基于视频时长的动态等距抽帧算法，兼顾解析细腻度与 Token 消耗。
  * **表情包识别**：可区分普通图片与表情包，优先按原消息链识别并注入提示词，帮助 LLM 明确当前内容是“表情包”而不是普通图片。
  * **文件感知**：当引用包含文件时，自动注入文件名；支持读取文本和 PDF 文件开头片段并注入，便于快速理解文件内容。
  * **卡片感知**：支持 JSON 卡片（如分享卡片/小程序）的关键信息提取，并可注入到上下文。
  * **URL 链接解析**：自动识别消息中的链接并注入网页标题与正文摘要；支持下载链接识别、超时/下载上限、内网拦截、域名黑名单与缓存控制。
  * **身份注入**：在解析引用消息或转发记录中的多媒体时，自动注入发送者昵称，帮助 LLM 准确分辨“谁发了什么”。
  * **智能屏蔽**：支持分别配置“引用 Bot 自身视频/文件/聊天记录”时的概率拦截，用于忽略用户对非 LLM 消息的回复。
  * **总结缓存**：自动缓存已处理视频的总结内容。当相同视频消息再次被引用时，直接复用缓存结果，提升响应速度并节省 token 消耗。
* **合并转发解析**：
  * **多层级解析**：支持完整读取合并转发记录中的文字内容，即使是多层嵌套的记录也能精准提取。
  * **媒体还原**：同步支持解析记录中的视频关键帧、图片与 JSON 卡片，且可配置视频处理数量与每个视频抽帧上限；任一上限为 0 时仅保留视频占位符。
  * **转发解析缓存**：会缓存“转发记录最终注入上下文”；相同 `forward_id` 且解析配置一致时直接复用缓存。

### 4️⃣ 🧩 信息注入：补齐场景与身份上下文

该模块包含两类注入能力：**环境感知注入** 与 **群成员信息注入**。

* 环境感知注入：
  * 可注入字段：发送时间、星期、节假日、时间段、平台、会话类型、群名、群号。
* 群成员信息注入：
  * 可注入对象：当前消息发送者 & Bot 自身。
  * 可注入字段：用户ID、昵称、群名片、性别、年龄、入群时间、最后发言时间、群等级、QQ等级、群身份、群头衔、地区、Q龄。

### 5️⃣ 🛠️ LLM 工具：赋予 Bot 现实行动力

Bot 可根据语境自主调用以下工具函数，无需人工干预，且部分工具支持跨会话指定群聊执行：

> 不使用的工具可在 AstrBot 的「管理行为」界面中直接禁用。

* 查询类：
  * **`get_user_avatar`**：获取指定用户头像。
  * **`get_contact_list`**：获取 Bot 通讯录。
  * **`get_group_members_info`**：获取群成员列表。
  * **`get_group_member_info`**：获取单个群成员详情。
  * **`get_group_info`**：获取群基础信息。
  * **`get_group_notices`**：获取群公告列表。
  * **`get_group_essence`**：获取群精华列表。
  * **`get_msg_history`**：按参数获取群聊或私聊历史消息（支持关键词与时间范围）。
* 群管理写操作：
  * **`send_message`**：按参数发送私聊或群消息。
  * **`set_group_ban`**：禁言/解禁成员。
  * **`kick_group_member`**：踢出单个成员。
  * **`set_group_whole_ban`**：开启/关闭全员禁言。
  * **`set_group_admin`**：设置/取消管理员。
  * **`set_group_card`**：设置群名片。
  * **`set_group_special_title`**：设置群头衔。
  * **`set_essence_msg`**：将引用消息设为精华。
  * **`delete_essence_msg`**：移出精华。
  * **`delete_msg`**：撤回消息。
  * **`set_group_name`**：设置群名称。
  * **`send_group_notice`**：发送群公告。
  * **`delete_group_notice`**：删除群公告。
  * **`set_group_kick_members`**：批量踢人。
  * **`dismiss_group`**：解散群。
* 黑名单工具：
  * **`block_user` / `unblock_user`**：拉黑或解除拉黑指定用户。
  * **`list_blacklist` / `get_blacklist_status`**：分页查询黑名单与单用户状态。

## 🔌 平台兼容说明

本插件默认按 AstrBot 通用事件工作，但部分能力依赖 QQ 协议适配（`aiocqhttp`）提供的 OneBot API。目前插件功能仅在 **NapCat** 环境完成实测。

* 除黑名单相关函数工具（`block_user`、`unblock_user`、`list_blacklist`、`get_blacklist_status`）外，其余 LLM 工具均仅 `aiocqhttp` 可用。
* 引用消息深度解析（含 `get_msg`、`get_forward_msg`）与合并转发内容展开。
* 动态合并与后续消息增强解析同样依赖 `get_msg` 回原消息链，当前用于统一解析合并中的图片/表情包、JSON、文件、URL 与视频。
* 撤回通知实时监控（`group_recall` / `friend_recall`）、请求前快速拦截、合并结束前可用性校验与动态重排后的兜底检查。
* 空 `@` 唤醒的“最近 5 条上文补充”依赖 `get_group_msg_history`。
* 群成员信息注入仅在 QQ 群聊（`aiocqhttp`）下生效。
