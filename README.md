## 📖 简介

一款为 [AstrBot](https://astrbot.app) 设计的 LLM 增强插件。通过集成智能唤醒、消息合并、多媒体解析、上下文感知、QQ 群成员信息注入与黑名单管理，并提供扩展的群管理 LLM 工具、跨会话定向发消息能力，以及可配置的管理员校验/危险工具开关/二次确认机制，提升 Bot 在群聊与私聊中的参与感、逻辑连贯性与安全性，让你的 Bot 不再只是被动响应，而是能够“听懂”话题、“看懂”视频、“感知”群成员并执行受控操作。

---

## ✨ 功能

本插件通过以下核心模块，将您的 Bot 升级为具备“直觉”与“多维感知”的智能实体：

* **🧠 智能唤醒系统**：支持概率唤醒、相关性、活跃期、特定提及等多种触发模式，内置节奏控制与并发限流机制。
* **📦 消息合并增强**：智能聚合短时间内多条消息，支持跨用户上下文合并、撤回实时监控、请求前快速拦截与兜底校验，以及合并数量上限控制。
* **🎬 媒体文件感知**：支持视频抽帧/ASR、JSON 卡片解析、文件引用感知与文本摘录注入、合并转发内容解析。支持聊天记录视频抽帧上限控制与占位符回退。内置视频总结缓存机制，避免重复解析。
* **🧩 信息注入增强**：提供独立的群聊上下文注入链路（可选主动唤醒前/全唤醒前），并支持环境感知、群成员信息注入以及通知/申请事件上下文记录，补齐场景和身份上下文。
* **🛠️ 增强管理工具**：集成通讯录查询、群成员查询、历史消息检索、群聊相关读写操作、跨会话消息发送、黑名单管理等 LLM 函数工具，形成“查询 -> 判断 -> 执行”的完整闭环，并支持管理员校验、危险工具开关与二次确认。
* **🛡️ 黑名单管理器**：支持拉黑/解除拉黑、分页查询与状态查询；支持三档黑名单拦截等级（仅 LLM / 指令+LLM / 所有消息）。

## 🚀 安装

1. **环境准备**：
   * 系统需安装 FFmpeg 并将其加入环境变量（用于视频/GIF 解析）。
   * 在插件目录执行 `pip install -r requirements.txt` 安装依赖。
2. **插件部署**：
   * 将 `astrbot_plugin_llm_enhancement` 文件夹放入 AstrBot 的 `plugins` 目录。
3. **重启 AstrBot**。

## 💡 使用

本插件旨在提供**全自动化**、**无感**的增强体验，以下是各模块详细的功能介绍与实现细节：

### 1️⃣ 🧠 智能唤醒：从“被动回复”到“主动参与”

Bot 不再死板地等待艾特，而是根据语境决定是否介入：

* **主动唤醒**：
  * **概率唤醒**：先累计一批普通群消息，再根据活跃度换算出观察门槛与命中概率进行一次抽样。活跃度越高，攒消息门槛越低、命中概率越高。
  * **空 @ 唤醒**：当用户空艾特时，插件会自动注入“某某@了你”提示，并补充该消息前最近 5 条群聊普通消息。
  * **提及唤醒**：支持 regex 规则。支持 `re:pattern` 与 `/pattern/flags`（flags 支持 `i,m,s,x,a,u,L`）；仅当规则既不以 `re:` 开头、也不符合 `/pattern/flags` 形式时，才按普通文本包含匹配。示例：`re:^(?=.*群主)(?=.*笨蛋).*$`。
  * **相关性唤醒**：自动分析当前消息与上下文的关联度。当聊到与上文相关的话题时，它可以主动跟进。并且会对超短消息、问句等场景做降噪；还可按当前消息长度动态选择历史条数。
  * **专业答疑**：识别群内专业提问或求助信号，精准提供帮助。答疑判定为“提问门槛 + 语义评分”两段式，减少非问句误唤醒。
  * **无聊唤醒**：识别“冷场/无聊/求聊天”语境并触发唤醒；采用前置门槛过滤长叙述，降低误触发。
  * **唤醒节奏优化**：对“提及他人/回复他人/多人并行对话”等典型非 Bot 对话信号做主动降权；同时对“用户复读相关性上下文片段”的场景做额外降权，并可在 Bot 长时间未参与话题时给予轻微沉默补偿。
* **唤醒判定**：
  * **前置唤醒判定**：可单独配置模型和提示词，仅对普通 `@` / 唤醒前缀请求生效，用于模拟真人沉默场景；支持占位符变量。
  * **后置唤醒兜底**：可指定“显式/概率/提及/相关性/答疑/无聊”等唤醒类型，在命中后再经过一次唤醒判定，进一步过滤误触发；当判定链路异常时会自动放行，避免误杀正常会话。
  * **活跃在线期**：Bot 回复后会进入短暂的“兴奋期”，此时用户的后续追问无需艾特即可直接响应。
  * **同用户延长**：可配置唤醒延长仅对“上次触发请求的用户”生效，避免窗口期被其他人消息打断。
  * **唤醒延长判定**：可选指定模型对“是否继续响应”做 T/F 判定；支持单独配置唤醒延长判定提示词，判定不可用时自动回退本地相似度。
* **约束限制**：
  * **并发控制**：支持用户级与群聊级控制；设置上限后，超限的新请求会在唤醒阶段前被拦截，以优化高并发场景下回复延迟显著的问题。
  * **唤醒屏蔽**：支持普通文本包含、正则规则与指令。命中后将阻止本次唤醒（Bot 管理员不受影响）。
  * **唤醒必须 @Bot**：开启后，群聊中仅通过唤醒前缀触发的 LLM 请求会被拦截，必须 @Bot 或引用 Bot 消息才放行，以实现进一步拟人。
  * **机器人消息跳过唤醒**：可配置机器人账号 ID，并分别指定跳过“显式唤醒”“主动唤醒”“唤醒延长”三类判定。

### 2️⃣ 📦 消息合并：聚合碎片输入

通过两种合并模式，解决碎片化输入导致的语义割裂与回复跳步问题：

* **模式选择**：
  * **动态软重算**：先发起请求；若响应返回前收到新消息，会放弃上一轮结果，并在下一轮重算上下文。
  * **硬等待合并**：在短窗口内先收集后续消息，再统一发起请求，有助于节省 token，但会导致平均响应时间变长。
* **窗口与边界控制**：
  * **统一规则**：两种模式都支持配置多人消息聚合、数量上限与后续消息唤醒要求，避免窗口期无限收集。
  * **撤回安全**：合并阶段支持实时撤回监听；命中后会立即剔除对应消息，必要时终止本次请求。
  * **最终校验**：请求与回复前会再次校验消息可用性；不可获取或已撤回消息会被过滤，全部失效时取消请求。

### 3️⃣ 🎬 理解增强：打破视觉次元壁

让 Bot 能看到听到富媒体消息的具体内容：

* **媒体消息感知**：
  * **私聊媒体唤醒**：私聊下直接发送官方表情/视频/文件/转发/语音/JSON 组件也可触发请求，无需额外文本。
  * **表情解析**：可区分普通图片与表情包，帮助 LLM 明确当前内容是“表情包”而不是普通图片，同时支持解析 QQ 黄脸表情含义。
  * **视频解析**：自动对视频进行抽帧与 ASR 语音转录，采用基于视频时长的动态等距抽帧算法，兼顾解析细腻度与 Token 消耗。
  * **语音解析**：支持语音消息 ASR 转写，注入上下文与引用解析，支持结果短期内缓存复用。
  * **文件解析**：当引用包含文件时，自动注入文件名；支持读取文本和 PDF 文件开头片段并注入，便于快速理解文件内容。
  * **卡片解析**：支持 JSON 卡片（如分享卡片/小程序）的关键信息提取，并可注入到上下文。
  * **链接解析**：自动识别消息中的链接并注入网页标题与正文摘要；支持下载链接识别、超时/下载上限、内网拦截、域名黑名单与缓存控制。
  * **合并转发解析**：支持完整读取合并转发记录中的文字内容，同步支持解析记录中的视频关键帧、图片与 JSON 卡片。
* **解析增强**：
  * **身份注入**：在解析引用消息或转发记录中的多媒体时，自动注入发送者昵称，帮助 LLM 准确分辨“谁发了什么”。
  * **智能屏蔽**：支持分别配置“引用 Bot 自身视频/文件/聊天记录”时的概率拦截，用于忽略群组成员对非 LLM 消息的回复。
  * **总结缓存**：自动缓存已处理视频/合并转发的总结内容。当相同消息再次被引用时，直接复用缓存结果，提升响应速度并节省 token 消耗。

### 4️⃣ 🧩 信息注入：补齐场景与身份上下文

该模块包含三类注入能力：**上下文注入**、**环境感知注入** 与 **群成员信息注入**。

* 上下文注入：
  * 提供独立的群聊上下文记录与注入链路，可选在主动唤醒前/全唤醒前/关闭。
  * 通知/申请事件（如撤回、管理变更、禁言、进退群、戳一戳、精华、入群申请）也会写入上下文。
  * 支持精简/详细两种注入格式；详细模式可携带 `msg_id`、`at/reply` 等结构信息，便于工具调用与后续推理。
  * 支持按配置解析 URL/JSON/语音/文件/聊天记录抽取等能力；图片转述与语音转写结果会进行短期缓存复用。
  * 支持通过 `/清除上下文` 指令（别名 `/clear`）快速清空当前群上下文。
* 环境感知注入：
  * 可注入字段：发送时间、星期、节假日、时间段、平台、会话类型、群名、群号。
* 群成员信息注入：
  * 可注入对象：当前消息发送者 & Bot 自身。
  * 可注入字段：用户ID、昵称、群名片、性别、年龄、入群时间、最后发言时间、群等级、QQ等级、群身份、群头衔、地区。

> [!TIP]
> * 建议关闭框架自带群聊感知，避免与本插件重复注入。  
> * 若开启上下文图像转述（无论框架或本插件），流程会等待视觉模型响应，整体时延可能增加。  

### 5️⃣ 🛠️ LLM 工具：赋予 Bot 现实行动力

Bot 可根据语境自主调用以下工具函数，无需人工干预，且部分工具支持跨会话指定群聊执行：

* 查询类：
  * **`get_user_avatar`**：获取指定用户头像。
  * **`get_contact_list`**：获取 Bot 通讯录。
  * **`get_group_members_info`**：获取群成员列表。
  * **`get_group_member_info`**：获取单个群成员详情。
  * **`get_group_info`**：获取群基础信息。
  * **`get_group_notices`**：获取群公告列表。
  * **`get_group_essence`**：获取群精华列表。
  * **`get_msg_history`**：按参数获取群聊或私聊历史消息（支持关键词与时间范围）。
* 群管理写操作：
  * **`send_message`**：按参数发送私聊或群消息。
  * **`set_group_ban`**：禁言/解禁成员。
  * **`kick_group_member`**：踢出单个成员。
  * **`set_group_whole_ban`**：开启/关闭全员禁言。
  * **`set_group_admin`**：设置/取消管理员。
  * **`set_group_card`**：设置群名片。
  * **`set_group_special_title`**：设置群头衔。
  * **`set_essence_msg`**：将引用消息设为精华。
  * **`delete_essence_msg`**：移出精华。
  * **`delete_msg`**：撤回消息。
  * **`set_group_name`**：设置群名称。
  * **`send_group_notice`**：发送群公告。
  * **`delete_group_notice`**：删除群公告。
  * **`set_group_kick_members`**：批量踢人。
  * **`dismiss_group`**：解散群。
* 黑名单工具：
  * **`block_user` / `unblock_user`**：拉黑或解除拉黑指定用户。
  * **`list_blacklist` / `get_blacklist_status`**：分页查询黑名单与单用户状态。

> [!TIP]
> 不使用的工具可在 AstrBot 的「管理行为」界面中直接禁用。

## 🤖 SubAgent 配置

若你准备为本插件注册的函数工具配置 SubAgent，下面是一套可供参考的基础方案。

| Agent 名称 | 人设提示词 | 对主 LLM 的描述 | 分配工具 |
| --- | --- | --- | --- |
| `group_read` | 你你是群与成员信息的只读查询助手。仅执行查询类操作，不做任何写操作或管理动作。整理查询结果并回显关键字段（群名、人数、成员身份等）。 | 负责通讯录/群/成员/历史消息的只读查询，也可查看用户头像。读类型工具（查看群公告、群成员等）原则上可自由调用，但需注意部分隐私性信息（如查看你的通讯录）不应展示给普通用户。 | `get_user_avatar`、`get_group_members_info`、`get_group_member_info`、`get_group_info`、`get_group_notices`、`get_group_essence`、`get_contact_list`、`get_msg_history` |
| `group_admin` | 你是群管理执行助手。只在用户明确授权时执行写操作。对高风险操作（如 dismiss_group）必须二次确认并回显目标群号与操作内容。若信息不完整，先追问确认。 | 只在用户明确授权时执行群管理写操作；若目标群组非当前会话且用户未提供群组ID，需先通过 group_read 获取。写类型工具（设置群精华、设置群公告等）根据用户群身份与请求合理性斟酌执行。踢人、全员禁言、改群管理结构、解散群等仅在必要且明确由管理员授权同意的场景执行。 | `set_group_ban`、`kick_group_member`、`set_group_whole_ban`、`set_group_admin`、`set_group_card`、`set_group_special_title`、`set_essence_msg`、`delete_essence_msg`、`delete_msg`、`set_group_name`、`send_group_notice`、`delete_group_notice`、`dismiss_group`、`set_group_kick_members` |
| `security_blacklist` | 你是黑名单与安全策略助手。使用“拉黑/解除拉黑”措辞。执行拉黑/解除拉黑前确认目标用户与期限（若有）。返回操作结果与生效时间。 | 执行拉黑/解除拉黑与黑名单查询；使用“拉黑/解除拉黑”措辞。在用户对你持续骚扰、持续恶意冒犯、换说法反复挑衅且屡教不改时可使用拉黑，用户玩笑性质的不合理/夸张请求无需拉黑。 | `block_user`、`unblock_user`、`list_blacklist`、`get_blacklist_status` |

> [!TIP]
> 建议开启 `主 LLM 去重重复工具` 开关。

## 🔌 平台兼容说明

目前以 **QQ（aiocqhttp / NapCat）** 为完整能力平台，其它平台可用通用能力，部分高级能力受限。

* **所有平台可用**：基础唤醒、消息合并、URL/文件/视频等常规增强（以框架的平台事件能提供的消息内容为准）。
* **Telegram 额外支持**：转发消息来源信息注入（例如“该消息转发自谁”）。
* **仅 QQ 可用**：大部分群管理类 LLM 工具、引用/合并转发深度展开、撤回实时监听与强一致拦截、空 `@` 自动补最近群聊上下文、群成员信息注入、QQ 黄脸表情解析、私聊场景的“正在输入”状态显示等。
* **黑名单工具**：跨平台可用（`block_user`、`unblock_user`、`list_blacklist`、`get_blacklist_status`）。

> [!IMPORTANT]
> 插件当前主要在 **NapCat** 环境实测通过；非 QQ 平台不保证可用性。
